# ☁️ Cloud Concepts, Types & Deployment Methods

---

## Q1. A company wants to use AWS for compute resources while keeping sensitive, legacy data in its own data center. Which deployment model should they use?

A) Public Cloud  
B) Private Cloud  
C) Hybrid Cloud  
D) Community Cloud  

<details>
<summary><strong>Show Answer</strong></summary>

**Correct Answer: C) Hybrid Cloud**

**Explanation:** A **Hybrid Cloud** combines public cloud services (like AWS) with on-premises infrastructure, allowing sensitive data to remain in the company’s data center while using cloud compute resources.

</details>

---

## Q2. Which deployment model offers the highest level of control, data privacy, and security, but is not cost-efficient for rapid scaling?

A) Public Cloud  
B) Private Cloud  
C) Hybrid Cloud  
D) Multi-cloud  

<details>
<summary><strong>Show Answer</strong></summary>

**Correct Answer: B) Private Cloud**

**Explanation:** A **Private Cloud** is dedicated to one organization and provides high control and security, but lacks elasticity and cost efficiency for rapid scaling.

</details>

---

## Q3. What is the main characteristic of the Public Cloud deployment model?

A) Infrastructure is owned by a single organization and managed on-premises.  
B) Resources are shared in a multi-tenant environment managed by a third-party provider.  
C) Data never leaves the company's own physical data center.  
D) It is only used by government agencies.  

<details>
<summary><strong>Show Answer</strong></summary>

**Correct Answer: B) Multi-tenant environment managed by a third-party provider**

**Explanation:** In a **Public Cloud**, infrastructure is owned and operated by a provider like AWS, and resources are shared across multiple customers.

</details>
